## 幻灯片 2 · 项目一览

## 幻灯片 7 · 建模 ②：无头 Blender 脚本管线

**方法论：不点鼠标，全用脚本**——可复现、可版本管理、可一键重跑。
- `rig_pipeline_v2.py`：导入 PBR、重塑腿、建骨架、刚性蒙皮 → 保存 `.blend`
- `animate_v2.py`：写 22 个动作进 NLA、渲染验证帧、导出 `character_v2.glb`
- 验证靠"看图"：每一步渲染 PNG 到 `verify/`，肉眼可查。
**坑点（真实踩过）**：中文版 Blender 按节点 `type` 找节点；Blender 5 渲染引擎名变化；动作是 slot-based。

## 幻灯片 8 · 绑定与蒙皮

**13 根骨骼**：Hips → Spine → Head → 左/右耳；Hips → 左/右臂、左/右腿、尾 Tail1→2→3→4。
**刚性蒙皮**：每个解剖部件 100% 绑到单根骨头（尾巴按 Y 位置分到 Tail1-4）。
**为什么刚性**：chibi 部件就是一截截"短桩"，整体移动即可，不需要权重过渡——简单、稳定、零穿插。
**约束**：手臂是横向短桩，X 轴抬升实测可到约 +70°（接近水平）不脱臂（旧记的"±20°"是误判）；竖直骨（Hips/Spine/Head）的"上"是 local-Y。

## 幻灯片 10 · 动作设计与制作 ②：原则与约束

**每个动作的节奏**：预备（anticipation）→ 保持（hold）→ 收势（settle），加耳朵/尾巴的次级运动让它"有重量、有惯性"。
**glTF 的硬约束**：导出会丢掉骨骼**缩放**动画——所以**没有挤压拉伸**，只能用平移+旋转表达力量感。
**接力规则**：循环动作（idle/walk/run/sleep）常驻，其余一次性播放完回到 idle。

## 幻灯片 11 · 表情系统：换贴图，不用骨头

**难点**：glTF 丢缩放，靠挤眼睛做表情在浏览器里完全无效；脸上也没有骨头。
**解法**：运行时**换头部材质的 baseColorTexture**——预生成 6 张脸图，按情绪即时切换、可短暂闪现再回到中性脸。
- 睁 / 眨 / 开心 / 委屈 / 惊讶 / 爱心 / 哭
- 触发：摇一摇=惊讶；对前置摄像头笑=爱心；连续 10+ 次抚摸=爱心；被冷落=哭。

## 幻灯片 13 · 技术管线总览

**一张图看懂数据流（脚本驱动，两条输出分支）：**
```
AI 生成 PBR 网格 (9 部件)
        │ rig_pipeline_v2.py
        ▼
蒙皮 .blend (13 骨 · 刚性)
        │ animate_v2.py (写 22 动作)
        ├─ export_print_v2.py        → STL (FDM/树脂)
        ├─ export_print_color_v2.py  → 全彩 OBJ/GLB
        ▼
动画 .blend
        ├─ animate_v2.py    → character_v2.glb → AR Web 应用
        └─ export_usd_v2.py → character_v2.usdz → iOS Quick Look
```
要点：AR 导出带 22 动作（动画 blend）；打印取静置几何融成单体（蒙皮 blend）——两家分叉，各取所需。

## 幻灯片 18 · 架构 · 成果 · 展望

**架构**（分层、模块化）
- 前端 `ar/`：CatRenderer 接口（three.js / model-viewer 双后端）+ CatModel（猫本体）+ ArSession 接口（MindAR / 绿色识别可换）+ VN/Story 模块。
- 后端 `worker/`：handlers / services / middleware / util 分层；`/api/chat-stream` 流式逐字回复。

**已完成**：全套建模/绑定/动画/表情/打印管线 + AR 为主的 VN 桌宠（P0–P4 完成）。
**展望**：P5 Blender 新增动作与表情（shy/bow/ponder/adore… + 腮红/思考/心动脸）、P6 收尾（配音/BGM/CG）。
