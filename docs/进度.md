## 踩过的坑 / 注意（含 CLAUDE.md 已记的）

- GLB 导出用 `AUTO` 贴图，**绝不 WebP**（安卓 Scene Viewer 拒收）；Draco 保留。
- 中文版 Blender：找节点按 `node.type`（如 `BSDF_PRINCIPLED`），不按显示名。
- glTF 丢动画**缩放**，所以动作只能平移+旋转，没有挤压拉伸；脸没有骨头，用换贴图。
- 硬骨架：手臂横向短桩，**X 轴抬升实测可到约 +70°（接近水平）不脱臂**（`arm_raise_test.py` 验证）。旧记的"±20°/+45° 上限"是**误判**，把所有抬手动作锁小了——"放不开"的元凶。审美上 emote 抬手 +30~50° 较自然（撒娇 +50° 偏夸张，已回调到 +28）。竖直骨（Hips/Spine/Head）的"上"是 local-Y。
- **整 360° 根骨旋转（spin / twirl / backflip）必须运行时重建**：glTF 导出会把它塌缩成"转到 180° 再退回"（四元数在 180° 反转），网页端表现为"转半圈倒回去 / 不连贯"。Blender 里看 Euler 是好的，骗人——只有看导出的 GLB 四元数轨道才发现。修复在 `ar/src/renderer/CatModel.ts` 的 `repairRootSpin`（原 `repairBackflip` 泛化版），三个旋转动作都要调。
- 彩色烘焙坑：color-only 烘焙只写 RGB、alpha 留 0 → glTF/EEVEE 预乘成黑；图像建 `alpha=False`/`alpha_mode='NONE'`，烘焙后**别再改 colorspace/alpha_mode** 否则缓冲被 generated_color 重建冲掉。
- `ar/` 是独立 git 仓（自己的远程），主仓 `.gitignore` 忽略它；改 AR 在 `ar/` 里提交。
- 摄像头需要安全环境（https/localhost），裸 LAN-IP http 不行。
